Introduction

ZohoDesk Embedded Functional Component

You may easily embed ZohoDesk functionality into your website in few minutes. Below section guides you to make integration easy

Introduction

Zohodesk EFC(Embedded Functional Component) is javascript SDK. It provides the following set of features/functionality:

    • Full Zoho Desk App
    • Tickets List
    • Ticket Detail
    • Headquarter Dashboard

As of now, the above features are available. The Quickstart section explains how to setup EFC in your website

Quick Start

        

/*snippet*/
<script>
        function ZOHODESK(ZD) 
          {
            ZD.setConfig({
              clientId: <client-id>,
              redirectUri: <redirect-url>,
              organizationName: <orgName>,
              container: <dom element>,
            });
          }
 </script>


<script 
      data-efc-locale="en_US" 
      src="https://js.zohostatic.com/support/build/zohostatic/zohodesk-efc-sdk-v1.0.0.js">
<script>

You can include the piece of JavaScript code into your website that will initialize Zohodesk into your website.

You have to create client using the following Link. This will give the clientId and secret key.




Note:

  • While create client choose "Client Type" as "javascript"
  • "ZOHODESK" method should be exist in the window object before loading "zohodesk-efc-sdk" js. EFC SDK calls "ZOHODESK" method and provides ZD Object. ZD Object has setConfig method. This method is used to initialize configuration for zohodesk.

Configuration Object

Configuration Object has following properties:

  • clientId - To identify integrator. creation of client id mention in the Quick Start section
  • redirectUri - You get the access token in the redirect url after hash. ex https://yourdomain.com/redirect#access_token=&scope=. You provide redirect url while creating client. Zoho accounts validate your credentials and send access token to redirect url. Redirect url should be same as in the client creation
  • organizationName - If you are mention this param it will render our EFC corresponding organization, If you not mention here it will be render the EFC based on your default org selection.
  • container - If you give the DOM element it will render Full deskApp in this DOM element
  • style - If you want customize zohodesk style you use this config param

        

        
          
                  {
                      clientId: ,
                      redirectUri: ,
                      (or)
                      getAccessToken:,
                      organizationName: ,
                      container: ,
                      style:,
                  }

Authentication

There are two ways to authenticate zohodesk

  • Client side Oauth2
  • getAccessToken method

OAUTH

If you don't want to handle getting access token then this approach is very useful to you. You don't need to think of oauth2 flow for getting access token. While render zohodesk it will open the popup and ask user credential. Once user provides him/her credential. It redirect to redirect url with access token. sdk get access token from redirect url. You must note redirect url must match with your domain. otherwise CORS error will throw sdk can't get access token.

  • Client Registration
  • Client Oauth Flow
  • Getting Access Token

Get AuthToken

In the above case, integrator won't worry about getting access token. But It is not suitable for all. Sometimes integrator shared common access token between all user. In the above configuration that was not possible. Instead you may provide javascript method to get the access token. That method should return a promise. Promise response should be following format.

         

        
            {
                expire:"1000*60*60",//one hour in milliseconds
                authToken:""
            }

Language Support

You can also possible to change the language for EFC. You can mention "data-efc-locale" param in script link tag. It loads "en_US" language as default. We are not providing any option to change language after sdk loaded. In future we may provide option to switch between languages.

         

          
<script

Zohodesk API methods

Methods

  • ZOHODESK.onload
  • ZOHODESK.routeTo
  • ZOHODESK.get




  • onload - This method will be trigger after our zohodesk sdk loaded into your website
  • routeTo - If you want the change view in EFC you can give the data via routeTo method.It will be change the view for EFC.
  • get - If you are want to get the information from our EFC you can use the ZOHODESK.get method.
             

            
  Get the Department Name

        ZOHODESK.get('routing.paramMap.deptName')




Set the URL param for ZohoDesk efc,


        some of the urls
        /agent/:sanitizedOrgname/:sanitizedDepartmentName/:sanitizedModuleName/(list|queue)/:sanitizedViewName
        ZOHODESK.routeTo(obj)



        Instead of create full url. we support some short url. example ticket detail view redirect format
        {
            name:"ticketdetail",
            id:""
        }

        common format
        {
            pathname,
            search,
            hash,
            state
        }

EFC Containers

Here the list of EFC containers,You have very easy to embed those containers into your websites

Tickets List

If you want render the Ticket List Items refer the below code snippet.It's working based on the queries that you are mention your code.You can refer the queries posibilities in this link. You can render our ListItems based on the department ID. 

         

        
ZOHODESK.onload(() => {

        let deptId = ZOHODESK.get(
        'departmentMapper.' + ZohoDesk.get('routing.paramMap.deptName')
      );


      ZOHODESK.renderContainer(
        'ListItems',
        {
              deptId: deptId,
              module: 'tickets',
              queries: [
                {
                  condition: ,
                  fieldName: ,
                  value: ,
                }
              ]
        },
        document.getElementById( DOM Element )
      );

)}

Ticket Detail

You can render our Ticket details view by using following code snippet. It's working based on the TicketId that you are given. Here also possible render the Ticket Peek View Also. 

         

        
  ZOHODESK.onload(() => {
        ZOHODESK.renderContainer(
            'TicketDetailview',
            {
              id: ,
              isPeekView: //optional
            },
            document.getElementById()
          );
  )}

Headquarters Report

If you want render the Headquarter report refer the below code snippet.

         

        
  ZOHODESK.onload(() => {

      ZohoDesk.renderContainer('HQDashboard', {}, document.getElementById());

    )}

Remove Container

If you want to unmount the our EFC from document you can use the below code

      

        
              ZOHODESK.removeContainer()
